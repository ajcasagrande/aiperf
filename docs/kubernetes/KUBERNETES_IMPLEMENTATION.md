## Architecture

### Deployed Components
- **8 Pods**: system-controller, timing-manager, records-manager, dataset-manager, worker-manager, worker(s), record-processor(s)
- **3 Kubernetes Services**: aiperf-system-controller, timing-manager, records-manager
- **1 Namespace**: Auto-generated with unique ID (auto-cleaned after completion)
- **RBAC**: ServiceAccount, Role, RoleBinding (auto-created)

### ZMQ Communication
```
System Controller (0.0.0.0:5661-5666) ← All services connect to proxies
   ↓
Timing Manager (0.0.0.0:5562-5563) ← Workers connect for credits
   ↓
Workers → Execute requests → Raw Inference Proxy
   ↓
Record Processors → Process metrics
   ↓
Records Manager (0.0.0.0:5557) ← Aggregates all records
```

### Result Retrieval
Results are retrieved from System Controller pod logs using special JSON markers:
```
<<<AIPERF_RESULTS_JSON_START>>>
{...full results JSON...}
<<<AIPERF_RESULTS_JSON_END>>>
```

The CLI extracts this JSON and displays results using ExporterManager.

## Usage

### Basic Usage
```bash
aiperf profile \
  --kubernetes \
  --url http://my-llm-service:8000 \
  --model my-model \
  --tokenizer gpt2 \
  --request-count 1000
```

### With Custom Namespace
```bash
aiperf profile \
  --kubernetes \
  --kubernetes-namespace my-benchmark \
  --no-kubernetes-auto-cleanup \
  --url http://my-llm-service:8000 \
  --model my-model \
  --tokenizer gpt2 \
  --request-count 1000
```

### All CLI Options
```
--kubernetes                          Enable Kubernetes mode
--kubernetes-namespace NAME           Custom namespace (optional, auto-generated by default)
--kubeconfig PATH                     Path to kubeconfig file
--kubernetes-image IMAGE              Container image (default: aiperf:latest)
--kubernetes-image-pull-policy POLICY Pull policy (default: IfNotPresent)
--kubernetes-service-account NAME     ServiceAccount (default: aiperf-service-account)
--kubernetes-auto-cleanup             Auto-cleanup resources (default: True)
--no-kubernetes-auto-cleanup          Disable auto-cleanup
```

## Troubleshooting

### No results displayed
- Check System Controller logs for JSON markers
- Verify pods completed successfully: `kubectl get pods -n <namespace>`
- Check for errors: `kubectl logs -n <namespace> system-controller-xxx`

### Pods stuck in Pending
- Check resources: `kubectl describe pod -n <namespace> <pod-name>`
- Verify image is available: `docker images | grep aiperf`

### Services not registering
- Check ZMQ connectivity: `kubectl logs -n <namespace> <service-pod>`
- Verify Kubernetes Services exist: `kubectl get svc -n <namespace>`
